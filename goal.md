## 1. 为什么无免费层？

这是个大胆的决定，但它是对的，前提是产品足够好。

**免费层的原罪：**
- 免费用户是成本，不是资产（服务器、AI 调用都要钱）
- 免费→付费转化率通常 2-5%，剩下 95% 都在烧你的钱
- 免费用户反馈往往偏离核心用户需求

**不做免费，但做试用：**

```
3 天全功能试用 → ¥1 锁定
  │
  ├─ 试用结束 → 自动扣费 ¥39/月
  └─ 随时可取消
```

- ¥1 不是收入，是筛选——愿意掏 1 块钱的人才会掏 39
- 3 天足够让用户体验 3 篇视频 + 3 次完整学习流程
- 试用结束时 AI 助手已经记住了 3 天的学习数据，用户舍不得丢

**定价：**

| 方案 | 价格 | 
|------|------|
| 月付 | ¥39/月（首月 ¥1） |
| 年付 | ¥299/年（首年 ¥239，约 ¥20/月） |

只有一档，买了全有。

## 2. 产品形态：粘贴链接为主，官方推送为辅

### 主流程（用户自己找内容）

```
用户粘贴 YouTube/B 站链接
  │
  ▼
系统处理（~2 分钟）
  ├─ 提取字幕（CC + Whisper）
  ├─ LLM 翻译
  ├─ 语法标注
  ├─ 难度评估
  └─ AI 出题
  │
  ▼
进入学习页面
  ├─ 视频播放 + 双语字幕
  ├─ 查词 + 跟读 + 练习
  └─ AI 助手记录
```

这是产品的心脏。用户想学什么就学什么——他追的美剧片段、关注的 YouTuber、TED 演讲、面试对话。自主权在用户手里。

### 辅流程（官方精选）

每天推送一篇精选视频（我们已经处理好的），给那些「今天不知道学什么」的用户。

### 为什么粘贴是核心？

Language Reactor 证明了「用户自带内容」是正确的模式：
- 用户找自己感兴趣的内容 → 学习动力远超被动接收
- 内容无限——YouTube 每天新增几十万条视频
- 我们不用背负内容生产的全部压力
- 用户粘性更高——「我的视频库」里有他自己的积累

官方推送解决另一个问题：**用户不知道从哪里开始的时候，我们给他一个起点。**

## 3. YouTube 为主 + 国内网络 = 自建视频库

这是两个看似矛盾的需求放在一起产生的必然结论。

### 问题

- YouTube 字幕质量最高（CC 覆盖率高、时间轴精准）
- 但中国用户无法直接访问 YouTube
- 嵌入 YouTube iframe → 国内用户看不了

### 解决方案：成品视频库

```
用户粘贴 YouTube 链接
  │
  ▼
香港服务器
  ├─ yt-dlp 下载视频 + 字幕
  ├─ Whisper 校正字幕
  ├─ LLM 翻译 + 语法标注
  ├─ 视频转码（多种清晰度）
  └─ 上传到 CDN（国内可访问）
  │
  ▼
视频存入我们的「成品视频库」
  ├─ 用户 A 贴过的链接 → 用户 B 再贴同一个 → 秒开（已处理过）
  └─ 库逐渐变大 → 越来越多的视频可以即贴即看
```

**关键设计：**

| 环节 | 方案 |
|------|------|
| 视频下载 | yt-dlp on 香港服务器 |
| 视频存储 | 阿里云 OSS（国内访问快） |
| 视频分发 | 阿里云 CDN（全国加速） |
| 字幕存储 | PostgreSQL（结构化，毫秒查询） |
| 已处理视频缓存 | 同一个 YouTube 链接不重复处理 |

### 版权问题

我们下载并存储视频 → 这在技术上涉及版权。

**我们的立场：**
- 视频由用户提交链接触发处理，我们是一个「工具」而非「内容平台」
- 类比：你用一个 App 下载了你想看的视频辅助学习，App 帮你处理了字幕
- 不公开索引用户提交的视频，不建内容广场
- 官方精选内容 → 走授权合作路线
- 长期 → 随着规模扩大，需要建立法务团队和授权体系

**现阶段实操：**
- 用户贴链接 → 处理 → 用户自己看（私人学习用途）
- 不对第三方公开用户视频
- 每个视频关联到提交它的用户

### 关键数据流

**用户贴链接处理流程：**
```
1. 用户提交 YouTube URL
2. 后端检查：这个链接处理过吗？
   ├─ 处理过 → 直接关联到用户视频库（秒返）
   └─ 没处理过 → 进入 Celery 队列
       ├─ yt-dlp 下载 (视频 + 最佳字幕轨道)
       ├─ FFmpeg 转码 (480p/720p/1080p)
       ├─ 上传 OSS
       ├─ Whisper 校正字幕时间轴
       ├─ LLM: 翻译 + 语法标注 + 难度 + 出题
       └─ 写入数据库 → 通知用户"可以开始学了"
3. 用户在视频库看到处理完成的视频
```

**跟读评分流程：**
```
1. 用户点击「跟读」→ 播放原声
2. 用户按住录音（或点击开始/停止）→ 浏览器 MediaRecorder → WebM 音频
3. 上传音频到后端
4. Whisper STT → 转文字
5. 文本比对：用户说的 vs 原文 → 差异分析
6. LLM: 基于差异 + 常见发音错误 → 中文反馈
7. 返回：评分 + 具体反馈 → 前端展示
```

## 9. 成为语言教学龙头的路径

```
Phase 1 — 工具（现在）
  Speaking = 最好的视频口语练习工具
  "想练口语？贴个链接开始。"

Phase 2 — 平台（有用户后）
  Speaking = 英语口语学习平台
  内容库丰富 → 个性化学习路径 → AI 教练闭环
  "每个人都有一个 AI 英语私教。"

Phase 3 — 生态（规模后）
  Speaking = 语言学习生态
  多语种（日语、韩语、西语...）
  创作者入驻 → 独家内容
  企业培训 → B 端收入
  "学任何语言，先在 Speaking 开口。"
```

英语 → 多语种 → 全球化。这条路很长，但每一步都是上一步的自然延伸。

## 12. 开发状态快照 (2026-05-30 更新)

- `docker-compose up -d` 启动全部服务（db + redis + backend + celery + frontend）

```
1. docker-compose up -d              # 启动所有服务
2. docker-compose exec backend alembic upgrade head  # 初始化数据库（首次）
3. 访问 http://localhost:3000
```
