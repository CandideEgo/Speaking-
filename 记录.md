# 视频评论质量评估系统 — 实施记录

## 完成时间
2026-06-10

## 背景
当前项目 Speaking 的视频选取标准依赖人工判断和简单的互动数据（点赞数、评论数），缺乏对评论内容质量的量化评估。本系统旨在通过提取真实 YouTube 评论并设计算法评估其质量，实现数据驱动的视频筛选。

## 评论质量评估算法

### 三维度评分系统

| 维度 | 权重 | 关键指标 |
|------|------|----------|
| **学习相关性** | 40% | 英语学习关键词出现频率（high/medium/low 三级关键词库） |
| **评论深度** | 30% | 平均长度、完整句子比例、提问比例、URL 分享比例 |
| **互动质量** | 30% | 平均点赞数、回复比例、高赞评论比例、评论者多样性 |

### 综合评分公式
```
overall_quality_score = learning_relevance_score * 0.4 + depth_score * 0.3 + engagement_score * 0.3
```

### 测试结果
- 高质量英语学习视频（TED talk）：67/100
- 低质量视频（spam 评论）：3/100
- 区分度：64 分差距

## 使用方式

### 1. 配置 YouTube API Key

在 [Google Cloud Console](https://console.cloud.google.com/) 创建项目并启用 YouTube Data API v3，获取 API Key。

```bash
# backend/.env
YOUTUBE_API_KEY=your-youtube-data-api-key
```

### 2. 运行数据库迁移

```bash
cd backend
alembic upgrade head
```

### 3. 触发分析

- **自动触发**：种子视频时自动触发（`seed_official_videos.py`）
- **手动触发**：调用 API

### 4. 查看结果

```bash
# 获取评论质量统计
curl http://localhost:8000/api/v1/comments/{video_id}/stats

# 获取高质量视频列表
curl "http://localhost:8000/api/v1/comments/top-videos?category=ted&limit=10"
```

---

## 注意事项

1. **YouTube API Quota**：每日 10,000 单位，每个 `commentThreads.list` 请求消耗 1 单位
2. **Cookie 问题**：yt-dlp 需要有效的 YouTube cookies 才能提取评论，建议优先使用 YouTube Data API
3. **数据隐私**：存储的评论数据仅用于质量分析，不对外展示完整评论内容

### 5. 生产环境注意事项

- **GPU 支持**：如果有 CUDA GPU，Whisper 会自动使用 GPU 加速（float16）
- **内存监控**：长视频分块转录时，确保有足够的内存和磁盘空间
- **Playwright 稳定性**：抖音反爬虫可能会变化，需要监控 Playwright 的稳定性
- **模型缓存**：首次运行会下载 Whisper 模型（base 约 150MB），建议预下载

### 6. 回滚方案

如果新转录流程出现问题，可以回滚到旧的 yt-dlp 字幕提取：

编辑 `backend/app/tasks/video_processing.py`，将 `_extract_subtitles` 恢复为原来的 yt-dlp 逻辑（代码已注释保留在文件中）。

### 7. 已知限制

- Whisper 转录质量低于 YouTube 官方字幕，准确性取决于音频质量和模型大小
- 抖音 Playwright 提取可能因网页结构变化而失效
- 长视频（>10分钟）转录时间较长，建议配置更大的 Celery 超时
- 无字幕视频现在可以处理，但转录结果可能包含识别错误
