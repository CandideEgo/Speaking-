### Phase 4 — 字幕识别错误修复 ✅
- **根因**（已确诊，非 Whisper）：`backend/app/services/ecdict.py` 的 `_parse_exchange` 把 ECDICT exchange 的 code `0`（原形反向指针）误当变形形式，建立错误反向映射 `inflected["good"]→"best"`、`["out"]→"outing"`、`["more"]→"mores"`。叠加 good/out/more 是 stopword（bnc≤100）被排除出主索引，`lookup` 命中错误的 inflected → 返回 best 的释义。
- **修复**：`_parse_exchange` 跳过 code `0`。回归测试 `test_parse_exchange_skips_lemma_code_to_avoid_wrong_reverse_mapping`。24 个相关测试全过。
- **降级机制确认**：gloss 端点有 AI 兜底（`ai.gloss_word_context`）——修复后 good 走 AI 实时查询返回语境释义，不再显示 best。
- **待用户操作**：重启 backend（索引重建）；可选跑 `backend/scripts/backfill_word_annotations.py` 重算已入库 `word_levels`。
- **遗留（未修）**：good/out/more 等 stopword 词，ECDICT 静态数据（音标/中译）因 stopword 过滤拿不到，只能靠 AI 兜底。stopword 是为字幕高亮不刷屏设的，但点词查询也共用此过滤。是否给点词放开 stopword 是另一个权衡，用户已叫停，未改。
