Watch 播放页重设计

三种布局原型 + skill 对比

所有原型都套用同一套"做减法"规则：只留 双语 / 英文 / 中文 三个字幕模式（听写/填空/翻译/词卡/阅读已砍掉）、字幕紧贴视频下方、跟读降为次要操作按需展开。横向对比的是布局而非功能。点击卡片打开对应原型。

A

视频全宽 + 下方标签页

视频几乎占满宽度，字幕紧贴下方，学习模式做成底部 Tab。视频最大化，结构最直观。
B

右栏可折叠（双列）

保留双列但视频列加宽，右栏字幕面板可一键收起、视频自动占满。兼顾常驻字幕与沉浸。
C

沉浸式居中

视频居中大尺寸，字幕叠加画面底部，字幕列表做成右侧滑入抽屉。最沉浸，但长字幕会遮画面。

skill 对比

A · 不用 skill

纯项目配色 token + 我的设计判断手写。GSAP 等动画 skill 对静态原型无用，未用。

A · 用 ui-ux-pro-max

同布局 A，叠加 skill 原则：focus-visible 键盘焦点、44px 触达、响应式 padding 分级、prefers-reduced-motion、color-shift hover。配色仍用项目橙（skill 自动推荐的 indigo+儿童字体不适配本应用，已弃用）。

选定一种布局（或提出合并/调整意见）后，我再进入真实代码重设计 —— 改 page.tsx / SubtitleModeTabs.tsx / globals.css / watchStore.ts，按项目规范先跑 gitnexus_impact 评估影响面。
